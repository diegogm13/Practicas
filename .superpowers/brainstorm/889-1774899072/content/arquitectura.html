¿Por qué Redis solo en el Gateway?

Cuando el usuario hace login, el Gateway guarda el token JWT en Redis con TTL de 24h. En cada petición posterior, el Gateway valida el token en Redis sin llamar al User Service — más rápido y menos carga. No se necesita Redis en los servicios individuales.

¿Por qué 3 bases de datos separadas?

Cada servicio tiene su propia BD en la misma instancia MongoDB. Así cada servicio es independiente — si mañana mueves Tickets a su propio servidor, solo cambias la connection string.
